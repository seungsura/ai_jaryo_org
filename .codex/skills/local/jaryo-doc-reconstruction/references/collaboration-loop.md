# Collaboration Loop

Use this loop whenever the goal is not just drafting, but improving quality with the user in the loop.

## Per-Pass Output

After each meaningful pass, return these four blocks:

1. `변경 내용`
   - What was improved or reorganized.
2. `아직 약한 부분`
   - Which claims, transitions, or examples still feel under-supported.
3. `사용자에게 물을 질문`
   - Ask 1 to 3 narrow questions only.
   - Each question should unlock a real editorial decision.
4. `추가 자료 요청 또는 흡수 계획`
   - If new material is needed, ask for a specific type of source.
   - If new material was already provided, say which target section it should change.

## Question Quality Rules

- Prefer specific editorial questions over broad brainstorming.
- Ask about one disputed fact, one missing example, or one structural choice at a time.
- Avoid asking the user to repeat information already present in the markdown.

## Source Integration Rules

When a new source arrives:

1. Identify its type: transcript, memo, outline, summary, or draft prose.
2. Extract the claims or examples worth preserving.
3. Map them to the target section or reference file.
4. Note contradictions with current drafts before editing.
5. Explain whether the source should:
   - strengthen an existing paragraph,
   - create a new subsection,
   - replace weak OCR-derived wording,
   - or remain in `docs/01-sources/intake/open-questions.md` until clarified.
