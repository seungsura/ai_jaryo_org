# Seminar Prose Tone Rules

## Target Voice

- canonical seminar prose는 기술 블로그 에세이처럼 읽혀야 한다
- 보고서, 안내문, 발표 대본, 강의안 말투로 미끄러지지 않는다
- 주장과 대비가 문단을 끌고 가야 하며, 안전한 요약문을 기본값으로 두지 않는다
- 강한 문장은 허용하지만 허세체나 밈체로 과장하지 않는다

## Preferred Characteristics

- 직접적이고 단정적인 서술
- paragraph 단위의 리듬과 압력
- 문제 제기 -> 대비 -> 결론으로 밀고 가는 구조
- 기술 용어는 English 유지, 나머지 문장은 살아 있는 한국어로 정리

## Avoid

- `상류/하류`, `강하게 호출된다`, `핵심은 ~ 데 있다`, `~의 측면에서`, `~라고 볼 수 있다` 같은 번역투
- 명사구만 무겁게 포개진 추상 문장
- bullet을 줄글로 옮긴 듯한 설명문
- 지나치게 점잖고 무해한 사내 문서체

## Warning Signs

- 문장이 맞기는 한데 말맛이 전혀 없음
- 영어 개념을 한국어로 기계적으로 치환한 흔적이 강함
- 모든 문단이 같은 리듬으로 평평하게 이어짐
- 주장보다 해설과 부연이 더 길어짐
