# Output Contract

Use these shapes when producing work with this skill.

## Section Map

- `Target file`
- `Source markdown`
- `Relevant page range or heading cluster`
- `Core claims to preserve`
- `Missing or ambiguous points`

## Section Draft

- Start with a concise heading that matches the target file.
- Write in coherent prose paragraphs, not bullet dumps.
- Preserve key numbers, comparisons, and named concepts from the source.
- End with a short `검토 필요` block only when material is incomplete or ambiguous.

## Editorial Review Packet

- `변경 내용`
- `아직 약한 부분`
- `사용자에게 물을 질문`
- `추가 자료 요청 또는 흡수 계획`

## Open Questions Entry

Use this format in `docs/01-sources/intake/open-questions.md`:

```md
## QNN. Short label
- 대상 섹션: <target file or section title>
- 부족한 내용: <what is missing or unclear>
- 왜 보류했는가: <why markdown evidence is insufficient>
- 사용자 확인 필요: <specific question to ask>
```

## Prohibited Output

- Do not cite JPG, PNG, or PDF evidence when the source policy is markdown-only.
- Do not fabricate speaker intent, chronology, or examples that are absent from the markdown.
- Do not leave raw OCR garbage in the final prose if a faithful rewrite is possible.
- Do not ask broad, low-signal questions when a narrow editorial question is enough.
