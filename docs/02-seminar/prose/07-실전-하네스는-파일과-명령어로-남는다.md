# 07. 실전 Harness는 파일과 명령어로 남는다

## 07-1. 실행 환경이 원칙을 대체해야 한다

실전에서 원칙이 기억으로만 남으면 오래가지 못합니다. 방향을 먼저 잡자, 검증을 앞에 두자, 역할을 나누자. 모두 맞는 말이지만 매번 사람이 기억해서 지켜야 한다면 금방 무너집니다.

그래서 실전 하네스는 새로운 도구를 더 붙이는 일이 아닙니다. 어떤 일은 누가 할지, 실패했을 때 어디서 멈출지, 증거는 어디에 남길지를 실행 환경으로 고정하는 일입니다.

## 07-2. 반복 지시는 파일로 고정한다

동일한 지시를 매 세션 반복하고 있다면, 그 지시는 대화창 밖으로 나가야 합니다. 공통 기준은 `CLAUDE.md`나 `AGENTS.md`에 둡니다. 팀이 공유해야 할 워크플로우는 `Skills`로 분리하고, 반드시 실행되어야 하는 규칙은 `Hook`으로 고정합니다.

Claude Code는 `CLAUDE.md`를 매 세션 시작 시 읽는 문서로 봅니다.[^07-claude-md] 다만 모든 것을 한 문서에 넣으면 기준이 아니라 소음이 됩니다. 항상 읽혀야 하는 기준은 짧게 두고, 필요할 때만 호출되는 절차는 skill로 분리하는 편이 좋습니다.

## 07-3. Hook와 명령어로 멈춤을 고정한다

실전에서는 `Plan`과 `Build`를 같은 흐름으로 돌리기보다, 쓰기 전 단계와 실행 단계를 구분하는 구조가 안정적입니다. 예측 가능한 실행은 `lint`, `type check`, `test`, `approval` 같은 명령어가 실패할 때 바로 멈추는 방식에서 만들어집니다.

`Hook`은 이벤트가 생길 때 동작합니다. 파일 수정 후 포맷 검사, 커밋 전 테스트, `.env` 변경 차단처럼 자동 실행되는 동작을 미리 박아 두면, "그럴듯해 보이니 넘어가자"는 순간을 줄일 수 있습니다.

Anthropic의 도구 설계 관점도 비슷합니다. 도구는 결정론적 장치와 에이전트 판단 사이의 계약처럼 동작합니다.[^07-agent-tools]

## 07-4. 암묵지는 규칙으로 바꾼다

리뷰에서 반복되는 판단은 팀의 자산이 됩니다. 그런데 그 판단이 채팅창에서 사라지면 다음 세션은 다시 새로 시작합니다. 매번 반복되는 포인트는 `Post-Mortem`에서 끝내지 말고, `CLAUDE.md` 갱신, `manifest`, `progress file`, `issue note` 같은 형태로 옮겨야 합니다.

작게는 `PROMPT.md`에 실수 항목을 한 줄씩 쌓는 방식도 가능합니다. 다음 실행에서 그 줄을 먼저 읽게 하면, 말로 한 피드백이 다음 루프의 규칙이 됩니다.

## 07-5. 이슈가 세션의 연속 상태가 된다

실무에서 제일 약한 고리 중 하나가 세션 기억입니다. 긴 작업을 채팅 로그에만 묶으면 중요한 결정이 중간에 묻힙니다. 그래서 작업 단위는 세션이 아니라 이슈로 두는 편이 안정적입니다.

이슈는 목적, 수용 기준, 현재 상태, 증거를 같이 들고 가야 합니다. 수용 기준에 대응하는 테스트 결과나 스크린샷, PR 링크를 붙여 두면 완료 판단도 같이 남습니다. 이렇게 하면 같은 이슈를 다시 열어도 시작점이 분명합니다.

`manifest`와 Git 이력은 무슨 결정을 왜 했는지 남기는 장치로 쓸 수 있습니다.[^07-manifest-paper]

## 07-6. 작업 공간을 물리적으로 분리한다

같은 창에서 탐색, 구현, 테스트 감시, 이슈 정리를 다 돌리면 컨텍스트가 뒤섞입니다. 실전에서는 탐색 창, 구현 창, 테스트 감시 창, 이슈/PR 창처럼 역할별 자리를 나누는 편이 안정적입니다.

`cmux` 같은 다중 터미널 구성과 `git worktree`는 이런 분리를 실제 작업 환경에 박는 방법입니다. 워크트리마다 파일 시스템을 분리하면 병렬로 작업해도 서로의 결과를 덮어쓰지 않습니다.

## 07-7. 첫 주 실전 구성은 작게 시작한다

한 번에 완전한 하네스를 만들 필요는 없습니다. 첫 주에는 최소 체인만 있어도 됩니다.

반복 기준 세 가지를 문서에 고정합니다. 위험 명령 하나를 차단 규칙으로 막습니다. 최소 필수 게이트 하나를 검사로 둡니다. 작업 경로는 이슈 단위로 나누고, 끝나면 의사결정 근거를 파일로 남깁니다.

규모가 커져도 결국 남는 것은 도구 과시가 아닙니다. 컨텍스트를 덜 잃고, 기준을 더 오래 유지하게 하는 구조입니다.
