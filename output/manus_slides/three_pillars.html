에이전틱 워크플로우의 3대 기둥

01 // INPUT

Source Boundary

입력층을 좁히고 근거의 층위를 명확히 구분합니다. 에이전트가 참조할 정보의 경계를 설정하여 오염을 방지합니다.

02 // LOGIC

Canonical Prose

대화창 밖으로 꺼낸, 흔들리지 않는 기준 문장을 세웁니다. 모든 산출물이 돌아와야 할 논리적 고정점입니다.

03 // OUTPUT

Design Contract

즉흥적 발명이 아닌, 약속된 문법 안에서의 생산을 유도합니다. 제약 조건이 에이전트의 생산성을 극대화합니다.
